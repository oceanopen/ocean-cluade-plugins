# Bold 模板包

本模板包将 `beautiful-html-templates` 的设计系统引入 `frontend-slides` 技能，而不会将其设为每份演示文稿的默认选项。

## 阅读指南

1. 首先阅读 `bold-template-pack/selection-index.json`。
2. 仅根据元数据筛选候选模板：
   - `mood`
   - `tone`
   - `best_for`
   - `avoid_for`
   - `formality`
   - `density`
   - `scheme`
3. 如需标题页预览，仅阅读相关候选模板的 `preview.md` 文件。
4. 用户选择 bold 模板后，仅阅读该模板的完整 `design.md`。
5. 不要阅读包中的每一个 `design.md`。
6. 除非所选的 `design.md` 缺少关键实现细节，否则不要阅读或复制源模板库中的 `template.html`。

完整的源元数据索引不包含在面向用户的技能中。正常生成流程应仅使用 `selection-index.json`。

## 如何在 Frontend Slides 中使用

预览组合：

- 1 个安全选项，来自 `STYLE_PRESETS.md`
- 至少 1 个大胆选项，来自本模板包
- 1 个 wildcard（百搭）选项，可以是本包中的另一个 bold 模板，也可以是自行生成的自定义设计

根据场景调整默认组合的基调：

- 对于董事会、法律、监管、医疗、投资者汇报或高度正式的内部演示，让安全选项非常克制，并选择更沉稳、正式度更高的 bold 模板。wildcard 应该感觉权威且具体，而非仅仅装饰性的。
- 对于大胆、编辑风、表现力强、实验性或高度设计化的演示，保持安全选项作为可读的后备方案，选择一个有力的 bold 模板，wildcard 可以是另一个冒险的模板，也可以是更好地匹配用户场合和氛围的自定义设计。

如果 wildcard 是自定义设计，它必须遵循 Frontend Slides 的高品质美学标准：独特的字体排版、坚定的配色方案、可识别的布局系统、与场景匹配的视觉创意、固定的 16:9 舞台行为，并且不显示任何过程标签，如"custom"、"wildcard"、"template"或"preview"。

## 实现契约

`design.md` 是设计系统的参考。将其视为风格配方，而非需要复制的内容。`preview.md` 仅是用于生成三个标题页选项的轻量风格卡片。

预览幻灯片必须是用户演示文稿的真实标题页。不要在幻灯片上渲染模板名称、选项标签、文件名、路径、`preview.md`、"generated from"或用户需求备注。

生成最终幻灯片时：

- 将 `frontend-slides` 输出保持为一个自包含的 HTML 文件。
- 包含 `viewport-base.css` 的完整内容。
- 将每个演示文稿生成为固定 1920×1080 的舞台，统一缩放到视口。即使源模板原本是视口自适应的，此规则同样适用。
- 将源 `design.md` 中的 `vw`、`vh` 和 `clamp()` 值视为设计比例，需转换为固定的 1920×1080 舞台坐标。
- 保留所选模板的字体、配色、装饰元素体系、间距节奏和组件语法。
- 保持用户的实际幻灯片内容为主体。模板风格应塑造呈现方式，而非覆盖信息或结构。
- 验证渲染输出是否存在文本溢出和面板重叠问题。一张卡片可能通过 `scrollHeight` 检查，但仍被另一个网格面板遮挡。
