CLAUDE.md — 项目级配置最佳实践
CLAUDE.md 的作用

CLAUDE.md 是项目根目录的配置文件，Claude Code 每次启动时自动加载。它相当于给 AI 助手的「项目说明书」，包含代码规范、架构说明和偏好设置。

  • 1定义代码风格和命名规范
  • 2说明项目架构和关键目录
  • 3列出测试命令和构建流程
  • 4声明常见陷阱和注意事项

## Tech Stack

提示词工程 — 提问的艺术

✓ 推荐做法

  • 指明具体文件和行号
  • 描述期望行为而非仅报错
  • 提供错误信息和上下文
  • 明确约束条件（语言、框架）
  • 分步骤描述复杂需求
  • 用 /init 生成初始 CLAUDE.md

✗ 避免做法

  • 模糊的指令（"修一下"）
  • 一次性提出多个不相关需求
  • 省略关键上下文信息
  • 要求一步完成超大重构
  • 忽视 AI 的澄清问题
  • 盲目批准所有操作
💡 核心原则：精确 > 冗长
好的提示词像给高级工程师的 Ticket：有背景、有目标、有约束。"修复 src/auth/login.ts:42 的类型错误，user 对象可能为 null，需要添加空值检查" 远比 "代码有 bug" 有效 100 倍。

进阶技巧
  • 1 使用 Hooks 自动化
    配置 pre/post hooks 自动执行格式化、lint 等操作，减少手动干预。
  • 2 善用 Memory 系统
    通过 /memory 保存项目偏好和用户习惯，跨会话保持上下文一致性。
  • 3 Worktree 隔离开发
    使用 worktree 在独立分支中工作，避免污染主分支，安全地并行探索。
  • 4 MCP Server 扩展
    连接外部 MCP 服务器，扩展 Claude Code 的能力边界（数据库、API、自定义工具）。
  • 5 管道与脚本集成
    将 Claude Code 嵌入 CI/CD 管道，实现自动化代码审查、文档生成等。
  • 6 图片驱动开发
    直接粘贴 UI 截图或设计稿，Claude Code 可分析视觉内容并生成对应代码。
  • 7 Compact 管理上下文
    长对话中使用 /compact 压缩历史，释放 token 空间给当前任务。
  • 8 Headless 模式
    非交互模式下运行 Claude Code，适合脚本化场景和自动化工作流。

常见误区与应对

✓ 正确心态

  • AI 是副驾驶，你是指挥官
  • 先审查理解，再批准执行
  • 小步迭代，频繁验证
  • 提供清晰的需求和约束
  • 保持对代码的掌控力
  • 把重复任务交给 AI

✗ 常见误区

  • 盲目信任所有 AI 输出
  • 一次性要求完成超大任务
  • 忽略安全审查直接批准
  • 不给上下文就让 AI 猜测
  • 完全依赖 AI 不理解代码
  • 跳过测试直接合并
⚠️ 黄金法则
永远不要合并不理解的代码。Claude Code 生成的代码应该像同事提交的 PR 一样经过你的审查。如果你无法解释代码在做什么，就要求 Claude 添加解释，或者自己先理解再合并。
